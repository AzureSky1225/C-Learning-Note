# **第四章：面向对象基础**



## 一、类与对象
### 一、类

#### **1.什么是类**

在 C++ 中，**类（class）**是一种**用户自定义的数据类型**。
它可以把：

* 数据
* 操作这些数据的函数

组织在一起，形成一个整体。

你可以把类理解成：

> **一个模板 / 一个蓝图**

这个模板规定了：

* 有哪些数据
* 有哪些功能
* 外部可以怎样使用它

---

#### **2.为什么要使用类**

类的主要作用有：

1. **封装数据和功能**
2. **提高代码可读性**
3. **提高代码复用性**
4. **提高代码可维护性**

如果不使用类，程序中相关的数据和函数可能会散落在各处；
而使用类之后，可以把它们统一组织起来，更像一个完整的“对象”。

---

#### **3.类的基本定义格式**

类的基本定义方式如下：

```cpp
class 类名
{
    // 访问权限
    // 成员变量
    // 成员函数
};
```

注意：

* 类定义结尾有一个分号 `;`
* 这是很多初学者容易漏掉的地方

---

#### **4.类中可以包含什么**

类中通常包含两类成员：

1. **数据成员**：变量，用来存储数据
2. **函数成员**：函数，用来实现功能

例如：

* `name`、`age`、`id` 是数据成员
* `speak()`、`playBall()` 是函数成员

所以你可以简单记成：

* **变量用来存储数据**
* **函数用来实现功能**

---

#### **5.类和对象的关系**

类本身更像一个“模板”或“设计图”，
真正根据这个模板创建出来的具体变量，叫做**对象（object）**。

例如：

```cpp
Student stu;
```

这里：

* `Student` 是类
* `stu` 是对象

可以这样理解：

* **类**：设计图、模板
* **对象**：根据设计图创建出来的具体实例

---

#### **6.类的示例代码**

```cpp
#include <iostream>
using namespace std;

class Student
{
    char name[20];
    int age;
    int id;

    void speak()
    {
        cout << "今年我" << age << "岁" << endl;
    }

    void playBall()
    {
        speak();
        cout << "打篮球" << endl;
    }
};

int main()
{
    Student stu;

    return 0;
}
```

---

#### **7.上面代码的逻辑分析**

这个类中定义了一个 `Student` 模板。

它有三项数据：

* `name`
* `age`
* `id`

还有两个功能：

* `speak()`：说出自己的年龄
* `playBall()`：先调用 `speak()`，再输出“打篮球”

在 `main()` 中：

```cpp
Student stu;
```

表示创建了一个 `Student` 类型的对象 `stu`

不过这里要注意：

* 这个版本没有显式写访问权限
* 对于 `class` 来说，默认访问权限是 `private`
* 所以这些成员默认都是私有的，类外不能直接访问

---

### 二、对象的创建与成员访问

#### **1.什么是对象**

对象是根据类创建出来的具体实例。

例如：

```cpp
Student stu;
```

这里：

* `Student` 是类名
* `stu` 是对象名

对象是真正用来存数据、调用功能的具体变量。

---

#### **2.对象的定义格式**

对象的定义格式如下：

```cpp
类名 对象名;
```

例如：

```cpp
Student stu;
Car myCar;
Book book1;
```

这表示：

* 创建了一个 `Student` 类型的对象 `stu`
* 创建了一个 `Car` 类型的对象 `myCar`
* 创建了一个 `Book` 类型的对象 `book1`

---

#### **3.对象如何访问成员**

如果类中的成员是 `public`，那么可以通过对象名和点号 `.` 来访问。

格式如下：

```cpp
对象名.成员名
```

例如：

```cpp
stu.name
stu.playBall()
```

这里：

* `stu.name` 表示访问对象 `stu` 的成员变量 `name`
* `stu.playBall()` 表示调用对象 `stu` 的成员函数 `playBall()`

---

#### **4.对象访问成员的示例代码**

```cpp
#include <iostream>
#include <cstring>
using namespace std;

class Student
{
public:
    char name[20];

    void speak()
    {
        cout << "大家好，我是" << name << endl;
    }
};

int main()
{
    Student stu;
    strcpy(stu.name, "张三");

    cout << stu.name << endl;
    stu.speak();

    return 0;
}
```

---

#### **5.上面代码的逻辑分析**

代码中：

```cpp
Student stu;
```

表示创建了一个 `Student` 对象 `stu`

然后：

```cpp
strcpy(stu.name, "张三");
```

给对象中的 `name` 赋值

再通过：

```cpp
cout << stu.name << endl;
stu.speak();
```

访问成员变量和成员函数。

---

### 三、类中成员和访问属性

#### **1.类中的成员**

类中的成员主要分为两种：

**1）数据成员**

也就是变量，用来保存对象的数据。

例如：

```cpp
char name[20];
int age;
int id;
```

---

**2）函数成员**

也就是成员函数，用来描述对象的功能。

例如：

```cpp
void speak()
{
    cout << "今年我" << age << "岁" << endl;
}
```

---

#### **2.访问属性（访问权限）**

在 C++ 中，类的成员可以通过访问权限进行控制。

常见的访问属性有三种：

1. `public`
2. `protected`
3. `private`

---

#### **3.public 公有成员**

`public` 表示公有成员，可以在：

* 类内部访问
* 类外部也直接访问

例如：

```cpp
public:
    char name[20];
```

如果 `name` 是公有的，那么类外就可以直接写：

```cpp
stu.name
```

---

#### **4.private 私有成员**

`private` 表示私有成员，只能在类内部访问，类外不能直接访问。

例如：

```cpp
private:
    int age;
```

这时类外不能直接写：

```cpp
stu.age
```

否则会报错。

通常把重要数据设为 `private`，是为了保护数据，不让外部随便修改。

---

#### **5.protected 保护成员**

`protected` 的特点是：

* 类内部可以访问
* 类外部不能直接访问

它和 `private` 在你目前这个阶段看起来很像。
真正的区别主要体现在**继承**中，后面学面向对象时会更清楚。

所以你现在可以先简单记：

* `public`：类外可直接访问
* `private`：类外不可直接访问
* `protected`：类外也不可直接访问，后面继承时会更有用

---

#### **6.class 的默认访问权限**

这一点很重要。

对于 `class` 来说，如果你不写访问权限，那么成员默认是：

```cpp
private
```

例如：

```cpp
class Student
{
    int age;
};
```

这里的 `age` 默认就是私有成员。

所以你可以记住：

* `class` 默认访问权限是 `private`
* `struct` 默认访问权限是 `public`

---

#### **7.为什么要设置访问权限**

访问权限的核心作用是：

> **封装**

封装的意思可以简单理解成：

* 把数据包起来
* 不让外部随便乱改
* 通过规定好的函数接口来访问数据

这样做的好处有：

1. 更安全
2. 更规范
3. 更方便后期维护
4. 可以在修改数据前先做检查

---

### 四、封装的意义

#### **1.什么是封装**

封装是面向对象编程中的一个重要思想。
它的核心可以简单理解为：

> **把数据和操作数据的函数放在一起，并控制外部访问方式**

也就是说：

* 不是所有数据都让外部直接修改
* 而是通过类提供的函数来访问和操作数据

---

#### **2.为什么要封装**

封装的主要作用有：

1. **保护数据，避免被随意修改**
2. **统一访问方式，使代码更规范**
3. **可以在修改数据前进行检查**
4. **提高代码的安全性和可维护性**

例如，一个人的年龄不应该被随便改成负数或者 1000 岁。
如果直接把 `age` 设为 `public`，外部就可以乱改；
如果把 `age` 设为 `private`，并通过 `setAge()` 修改，就可以先检查是否合法。

---

#### **3.典型做法：数据私有，函数公开**

在实际设计中，很常见的一种思路是：

* 数据成员设为 `private`
* 成员函数设为 `public`

然后通过公开函数来操作私有数据。

例如：

* `setAge()`：设置年龄
* `getAge()`：获取年龄

这样外部就不能直接改 `age`，而必须通过函数来操作。

---

### 五、set 和 get 函数

#### **1.什么是 set 和 get**

在类中，经常会写两类函数：

* `setXxx()`：设置某个成员变量
* `getXxx()`：获取某个成员变量

例如：

```cpp
void setAge(int x)
{
    age = x;
}

int getAge()
{
    return age;
}
```

---

#### **2.set 和 get 的作用**

它们的主要作用有：

1. **代替类外直接访问私有成员**
2. **让代码更符合封装思想**
3. **可以在设置时加入检查逻辑**
4. **可以统一数据访问方式**

---

#### **3.为什么不用直接公开数据成员**

如果把 `age` 直接写成：

```cpp
public:
    int age;
```

那么外部就可以随便写：

```cpp
stu.age = -100;
```

这显然不合理。

而使用 `setAge()` 后，就可以在函数里做检查，避免非法数据。

---

#### **4.set 和 get 示例代码**

```cpp
#include <iostream>
using namespace std;

class Student
{
private:
    int age;

public:
    void setAge(int x)
    {
        if (x >= 0 && x <= 150)
        {
            age = x;
        }
    }

    int getAge()
    {
        return age;
    }
};

int main()
{
    Student stu;
    stu.setAge(20);

    cout << "年龄：" << stu.getAge() << endl;

    return 0;
}
```

---

### 六、类中成员函数的特点

#### **1.成员函数可以直接访问成员变量**

类中的成员函数，可以直接使用类中的成员变量。

例如：

```cpp
void speak()
{
    cout << "今年我" << age << "岁" << endl;
}
```

这里的 `speak()` 可以直接访问 `age`，因为它们都属于同一个类。

---

#### **2.成员函数之间可以互相调用**

同一个类中的成员函数之间也可以直接互相调用。

例如：

```cpp
void speak()
{
    cout << "今年我" << age << "岁" << endl;
}

void playBall()
{
    speak();
    cout << "打篮球" << endl;
}
```

这里 `playBall()` 里直接调用了 `speak()`

---

#### **3.成员函数可以带参数，也可以不带参数**

例如：

不带参数：

```cpp
void speak()
{
    cout << "hello" << endl;
}
```

带参数：

```cpp
void setAge(int x)
{
    age = x;
}
```

---

#### **4.成员函数也可以有返回值**

例如：

```cpp
int getAge()
{
    return age;
}
```

表示函数执行后，会把 `age` 返回出去。

---

### 七、完整示例：Student 类

#### **1.示例代码**

```cpp
#include <iostream>
#include <cstring>
using namespace std;

class Student
{
private:
    int age;
    int id;

public:
    char name[20];

    void setAge(int x)
    {
        if (x >= 0 && x <= 150)
        {
            age = x;
        }
    }

    int getAge()
    {
        return age;
    }

    void speak()
    {
        cout << "今年我" << age << "岁" << endl;
    }

    void playBall()
    {
        speak();
        cout << "打篮球" << endl;
    }
};

int main()
{
    Student stu;

    strcpy(stu.name, "张三");
    stu.setAge(18);

    cout << stu.name << endl;
    cout << "年龄：" << stu.getAge() << endl;
    stu.playBall();

    return 0;
}
```

---

#### **2.上面代码的逻辑分析**

这个类中：

* `age` 和 `id` 是私有成员
* `name` 是公有成员
* `setAge()` 用来设置年龄
* `getAge()` 用来获取年龄
* `speak()` 用来输出年龄
* `playBall()` 会先调用 `speak()`，再输出“打篮球”

在 `main()` 中：

```cpp
Student stu;
```

表示创建一个学生对象 `stu`

然后：

```cpp
strcpy(stu.name, "张三");
```

给公有成员 `name` 赋值

再通过：

```cpp
stu.setAge(18);
```

设置私有成员 `age`

最后输出姓名和年龄，并调用：

```cpp
stu.playBall();
```

---

#### **3.为什么 `age` 不能直接访问**

因为 `age` 是：

```cpp
private:
    int age;
```

所以类外不能直接写：

```cpp
stu.age = 18;
```

这会报错。

只能通过公开的函数来操作，比如：

```cpp
stu.setAge(18);
```

这就是封装的体现。

### 八、类设计时的基本建议

#### **1.一个类尽量描述一种对象**

类应该围绕一个明确主题来设计。

例如：

* `Student` 类描述学生
* `Car` 类描述汽车
* `Book` 类描述书本

不要把完全无关的内容都塞进同一个类里。

---

#### **2.类中的数据和行为应尽量相关**

例如学生类中可以有：

* 姓名
* 年龄
* 学号
* 说话
* 学习
* 打球

这些都是和“学生”这个对象有关的内容。

但如果突然在 `Student` 类里写“计算圆面积”，就不太合适。

---

#### **3.不要定义过多无关成员**

一个类如果成员太多、职责太杂，会导致：

* 可读性变差
* 可维护性变差
* 结构混乱

所以设计类时要尽量保持：

> **职责明确，结构简洁**

---

#### **4.重要数据尽量私有化**

如果某个数据不希望被类外随便修改，就应该优先考虑设为 `private`。

例如：

* 年龄
* 学号
* 分数
* 余额

然后通过公开函数来访问和修改。

---

#### **5.成员变量最好尽快赋值**

如果类中的成员变量定义后没有赋值，直接使用时可能会出现不确定结果。

例如：

```cpp
class Student
{
private:
    int age;
};
```

这里的 `age` 如果没有设置就直接使用，值通常是不确定的。

所以：

* 要么创建对象后尽快赋值
* 要么以后学构造函数时再统一初始化

---

#### **6.char 数组作为名字的局限性**

你现在类里经常写：

```cpp
char name[20];
```

这种写法在当前阶段可以用，但要注意：

* 长度固定
* 使用时要注意不要超过数组大小
* 以后学到 `string` 后，处理字符串会更方便

---

例如：

```cpp
int main()
{
    Student stu;

```

这里的 `stu` 是 `main()` 中定义的局部对象，
它只能在 `main()` 函数内部使用。

#### **2.class 和 struct 的一个区别**

初学阶段可以先记住一个最直接的区别：

* `class` 默认访问权限是 `private`
* `struct` 默认访问权限是 `public`

例如：

```cpp
class A
{
    int x;
};
```

这里的 `x` 默认私有。

```cpp
struct B
{
    int x;
};
```

这里的 `x` 默认公有。

---

#### **3.类定义最后不要忘记分号**

例如：

```cpp
class Student
{
};
```

最后这个分号必须有。

这是类定义中非常容易漏掉的一个点。

### 十一、容易混淆的点

#### **1.类和对象不是一回事**

* 类：模板
* 对象：具体实例

---

#### **2.private 不是“不能用”，而是“不能在类外直接用”**

例如：

* 在类的成员函数里可以访问 `age`
* 但在 `main()` 里不能直接访问 `stu.age`

---

#### **3.成员函数不是普通函数**

成员函数属于类，可以直接访问类中的成员变量。

---

#### **4.设置引用式接口不等于改绑**

`setAge()` 改变的是成员变量的值，
而不是把成员“换成别的东西”。

## 二、面向对象

#### **1.什么是面向对象**

C++ 是一种支持**面向对象编程（Object-Oriented Programming，OOP）**的语言。

面向对象是一种编程思想，也是一种编程范式。
它强调：

> **把程序中的数据和操作这些数据的方法组织在一起，形成对象来处理问题**

也就是说，在面向对象编程中，我们不再只关心“要写哪些函数”，
还会关心：

* 现实中的这个事物有哪些属性？
* 它有哪些行为？
* 应该把哪些数据和功能放在一起？

例如：

* 学生有姓名、年龄、学号
* 学生会说话、学习、打球

那么我们就可以把“学生”设计成一个类，再创建具体对象。

---

#### **2.为什么要使用面向对象**

面向对象的主要优点有：

1. **更符合人们理解现实世界的方式**
2. **更容易组织复杂程序**
3. **提高代码的可读性**
4. **提高代码的复用性**
5. **提高代码的可维护性和扩展性**

例如在一个大型程序中，如果只用很多散乱的变量和函数来写，代码会越来越乱；
而面向对象会把相关数据和行为打包起来，使结构更清晰。

---

#### **3.面向对象中的几个核心概念**

C++ 中常见的面向对象概念有：

1. **类（class）**
2. **对象（object）**
3. **封装（encapsulation）**
4. **继承（inheritance）**
5. **多态（polymorphism）**

---

### 1）类

类是一种用户自定义的数据类型，可以把：

* 数据
* 操作数据的函数

组织在一起。

可以把类看作一个模板或蓝图。

例如：

```cpp
class Student
{
    // 成员变量
    // 成员函数
};
```

---

### 2）对象

对象是类的具体实例。
也就是说：

* 类是模板
* 对象是根据模板创建出来的具体东西

例如：

```cpp
Student stu;
```

这里：

* `Student` 是类
* `stu` 是对象

---

### 3）封装

封装的意思是：

> **把数据和操作数据的函数放在一起，并控制外部访问方式**

例如把年龄 `age` 设置为 `private`，
再通过 `setAge()` 和 `getAge()` 来访问。

封装的好处有：

* 保护数据
* 避免乱改
* 提高安全性
* 方便后期维护


---

### 4）继承

继承是一种创建新类的方式，可以在原有类的基础上继续扩展。

你可以把它理解成：

> **“子类”继承“父类”的已有属性和功能，再增加自己的新内容**

例如：

* 有一个类 `Animal`
* 它有一些通用属性和行为
* 然后再派生出 `Dog`、`Cat`

这样就不用把相同代码重复写很多遍。

继承的主要作用是：

1. **代码复用**
2. **减少重复**
3. **更方便扩展新功能**

---

### 5）多态

多态的字面意思是：

> **同一种接口，表现出不同的行为**

简单理解就是：

* 不同子类对象
* 可以以一种统一方式来使用
* 但实际执行的行为可以不同

例如：

* 父类中有一个 `speak()` 方法
* `Dog` 的 `speak()` 可能输出“汪汪”
* `Cat` 的 `speak()` 可能输出“喵喵”

调用方式看起来一样，但结果不同，这就是多态的核心思想。

> **多态让不同对象在统一接口下表现出不同的行为**

后面真正学到虚函数时再展开。

---

#### **4.面向对象的核心思想总结**

可以把面向对象简单概括成一句话：

> **把现实中的事物抽象成类，再通过对象来组织程序**

例如：

* 学生 → `Student`
* 汽车 → `Car`
* 书本 → `Book`

每个对象都有：

* 自己的数据（属性）
* 自己的功能（行为）

---

#### **5.面向对象和面向过程的简单区别**

可以先有一个大概印象。

### 面向过程

更关注：

* 先做什么
* 再做什么
* 程序按步骤怎么走

它更像是在写“做事流程”。

---

### 面向对象

更关注：

* 有哪些对象
* 每个对象负责什么
* 数据和功能如何绑定在一起

它更像是在组织“参与做事的角色”。

---

#### **6.一个简单理解例子**

如果写“学生打球”：

### 面向过程可能会想：

* 定义年龄变量
* 定义姓名变量
* 写一个函数打印学生信息
* 写一个函数打印打球动作

### 面向对象会想：

* 创建一个 `Student` 类
* 里面放 `name`、`age`
* 再放 `speak()`、`playBall()`

这样程序结构会更自然。

---

#### **7.面向对象的意义**

面向对象并不是为了“显得高级”，而是为了：

* 更自然地建模现实问题
* 更方便组织复杂代码
* 更方便多人协作
* 更便于后期修改和扩展

---

## 三、编程原则

#### **1.什么是编程原则**

编程原则是指在写程序时，为了得到：

* 更高质量的代码
* 更清晰的结构
* 更好的可维护性
* 更强的扩展性

而遵循的一些基本思想和经验规则。

这些原则不一定是“语法强制要求”，
但它们能帮助我们写出更好的程序。

---

#### **2.为什么要学习编程原则**

初学时，代码能跑就已经很好了；
但如果代码越来越复杂，只追求“能跑”，以后就会出现很多问题，比如：

* 代码难看懂
* 功能难修改
* 到处重复
* 一改就容易出 bug

编程原则的作用就是帮助我们尽量避免这些问题。

---

### 一、常见编程原则

---

#### 1. DRY 原则

 **1）什么是 DRY**

DRY 是：

```text
Don't Repeat Yourself
```

意思是：

> **不要重复你自己**

也就是说，在程序设计中，应尽可能避免重复代码，
而是通过抽象和封装来复用代码。

---

**2）为什么要遵守 DRY**

如果同样的代码写了很多遍，会有这些问题：

1. 难维护
2. 容易漏改
3. 容易出错
4. 代码会越来越臃肿

例如：

如果在很多地方都写相同的“年龄合法性判断”，
那以后规则改了，你就得一处处去改。

这时更好的做法就是：

* 写成函数
* 或者封装到类里统一处理

---

 **3）DRY 的简单例子**

不好的写法：

```cpp
if (age >= 0 && age <= 150)
{
    // ...
}
```

在很多地方都重复出现。

更好的做法是：

* 写成函数
* 或者放进 `setAge()` 这样的成员函数里统一处理

---

#### 2. KISS 原则

 **1）什么是 KISS**

KISS 是：

```text
Keep It Simple and Stupid
```

更常见的理解是：

> **保持简单**

也就是说，在程序设计中，应尽可能让代码：

* 简洁
* 清楚
* 容易读懂
* 不要故意写复杂

---

**2）为什么 KISS 很重要**

代码不是只给电脑看的，也是给人看的。
如果一段代码虽然很“聪明”，但特别难看懂，那以后维护会非常痛苦。

所以通常来说：

> **能简单解决的问题，就不要故意绕得很复杂**

---

 **3）KISS 的简单例子**

例如明明一个 `if...else` 就能解决的问题，
就不要硬写成非常绕的嵌套表达式。

简单、直白、清楚，通常更好。

---

#### 3. YAGNI 原则

**1）什么是 YAGNI**

YAGNI 是：

```text
You Aren't Gonna Need It
```

意思是：

> **你现在还不需要它**

也就是说，不要过早加入还用不上的功能，不要为了“以后可能会用”而提前写一大堆复杂设计。

---

**2）为什么要注意 YAGNI**

很多初学者或者开发者会有一种倾向：

* 这个功能现在还没用到
* 但我先写上吧，万一以后会用呢？

结果就是：

* 代码变复杂
* 实际上根本没用上
* 维护成本变高

所以更好的思路是：

> **先实现当前真正需要的功能，再逐步扩展**

---

**3）YAGNI 的简单理解**

比如现在只需要做一个简单学生类，就先写：

* `name`
* `age`
* `speak()`

就够了。

不要一开始就加上：

* 100 个你还没用到的成员变量
* 一大堆未来根本不一定需要的函数

---

#### 4. 高内聚、低耦合原则

**1）什么是高内聚**

高内聚是指：

> **一个模块、一个函数、一个类，最好只负责完成比较单一、明确的功能**

例如：

* `Student` 类主要描述学生
* `Book` 类主要描述书本
* `setAge()` 就只做设置年龄这件事

这就叫高内聚。

---

**2）什么是低耦合**

低耦合是指：

> **不同模块、不同类之间的依赖关系越少越好**

也就是说，一个类不要和太多别的类死死绑在一起。

如果一改 A，B、C、D 全都得跟着改，那耦合就太高了。

---

**3）为什么高内聚低耦合重要**

如果一个类职责太杂，会出现：

* 很难理解
* 很难维护
* 很难修改

如果类和类之间依赖太紧，会出现：

* 改一点就牵一大片
* 程序越来越难扩展

所以理想状态是：

* **一个类内部功能尽量集中**
* **类和类之间尽量少耦合**

---

#### 5. SOLID 原则

**1）什么是 SOLID**

SOLID 是面向对象设计中常提到的一组原则缩写，包括五条：

1. **单一职责原则（SRP）**
2. **开放封闭原则（OCP）**
3. **里氏替换原则（LSP）**
4. **接口隔离原则（ISP）**
5. **依赖倒置原则（DIP）**

这五条原则比较偏设计层面，初学阶段不用死背细节，
但可以先知道它们在强调“怎样设计更好的类和模块”。

---

**2）单一职责原则（SRP）**

SRP 是：

```text
Single Responsibility Principle
```

意思是：

> **一个类或模块最好只负责一件主要事情**

例如：

* `Student` 类就主要描述学生
* 不要同时负责“学生信息管理 + 网络通信 + 数据库存储 + 打印报表”

否则职责太多，类会变得臃肿混乱。

---

**3）开放封闭原则（OCP）**

OCP 是：

```text
Open Closed Principle
```

意思是：

> **对扩展开放，对修改关闭**

简单理解就是：

* 新需求来了，尽量通过扩展来实现
* 而不是老去大改原来的代码

这样可以减少对旧代码的破坏。

---

**4）里氏替换原则（LSP）**

LSP 是：

```text
Liskov Substitution Principle
```

意思是：

> **子类对象应该能够替代父类对象使用，而不破坏程序原有逻辑**

这个原则和继承、多态关系比较紧密，
你现在先知道它在强调“继承关系要合理”就够了。

---

**5）接口隔离原则（ISP）**

ISP 是：

```text
Interface Segregation Principle
```

意思是：

> **不要让一个类依赖它不需要的接口**

简单理解就是：

* 接口不要设计得又大又杂
* 应该尽量小而专一

---

**6）依赖倒置原则（DIP）**

DIP 是：

```text
Dependency Inversion Principle
```

意思是：

> **高层模块不应该依赖低层模块的具体实现，而应该依赖抽象**

这个原则在你现在阶段可以先不深究，
知道它是面向对象设计中的重要原则即可。

---

### 二、当前阶段最值得先掌握的原则

如果结合现在的学习阶段，最值得先真正记住并实践的，其实是下面这几个：

1. **封装**
2. **DRY**
3. **KISS**
4. **YAGNI**
5. **高内聚、低耦合**
6. **单一职责原则（SRP）**

因为这些最容易立刻体现在写的代码里。

---

### 三、结合当前学习阶段的理解



**1.封装**

把 `age` 设为 `private`，再通过 `setAge()` / `getAge()` 访问。

**2.DRY**

不要在很多地方重复写相同代码，可以抽成函数或类。

**3.KISS**

能写简单就不要故意写复杂。

**4.YAGNI**

先写当前真正需要的成员变量和成员函数，不要一开始就把类设计得过于庞大。

**5.高内聚**

`Student` 类就尽量只管学生相关内容。

**6.低耦合**

不要让 `Student` 类无缘无故和大量不相关类绑得很紧。

---

### 四、面向对象与编程原则的关系

可以简单理解成：

* **面向对象**：是一种组织程序的思想
* **编程原则**：是帮助我们把这种思想用得更合理的一套规则

也就是说：

* 类、对象、继承、多态，告诉你“可以怎么写”
* DRY、KISS、SOLID 等原则，告诉你“怎样写得更好”

---

### 五、总结

#### **1.面向对象**

* 面向对象是一种编程思想
* 它强调把数据和操作数据的方法组织在一起
* 核心概念包括：

  * 类
  * 对象
  * 封装
  * 继承
  * 多态

#### **2.编程原则**

* 编程原则是写高质量代码时应遵循的一些经验规则
* 常见原则包括：

  * DRY：不要重复代码
  * KISS：保持简单
  * YAGNI：不要过早设计未来不需要的功能
  * 高内聚、低耦合：职责集中，依赖尽量少
  * SOLID：面向对象设计中的重要原则集合

#### **3.当前阶段建议**

* 先把“封装、DRY、KISS、YAGNI、高内聚低耦合”理解并用在日常代码里
* 继承、多态、SOLID 的更深层内容，后面随着类的学习再逐步展开

---

### 六、初学阶段最容易混淆的点

#### **1.面向对象不等于只是“会写类”**

会定义类只是开始，
真正的面向对象还包括：

* 怎么设计类
* 怎么组织职责
* 怎么做封装
* 怎么让代码更清晰

---

#### **2.继承和多态现在不用强行死背细节**

现在更重要的是先把：

* 类
* 对象
* 成员变量
* 成员函数
* 访问权限
* 封装

这些基础打牢。

---

#### **3.编程原则不是“硬性语法”**

例如：

* 不遵守 DRY，程序也许照样能跑
* 不遵守 KISS，程序也许也能跑

但代码会越来越难维护，所以这些原则是帮助写“更好代码”的。
